# Mini Stock System — Product Management Module
## Functional Requirements (Full Detail)

---

## 1. จัดการสินค้า (CRUD)

### 1.1 เพิ่มสินค้าใหม่
- กรอกข้อมูลสินค้าผ่านฟอร์ม
- ระบบ auto-generate รหัสสินค้า (SKU) หรือกรอกเองก็ได้
- ตรวจสอบ SKU ซ้ำแบบ real-time ก่อน save
- อัปโหลดรูปภาพได้หลายรูป (max 5 รูป, 5MB/รูป, JPG/PNG/WebP)
- กำหนดหน่วยนับของสินค้า (ชิ้น, กล่อง, โหล ฯลฯ)
- ตั้งค่า stock ขั้นต่ำ (min stock) สำหรับแจ้งเตือน
- บันทึกรายละเอียด/หมายเหตุเพิ่มเติมแบบ free-text

### 1.2 แก้ไขสินค้า
- แก้ไขข้อมูลทุก field ได้ ยกเว้น SKU (เปลี่ยนไม่ได้หลัง save)
- ระบบบันทึก log ทุกครั้งที่มีการแก้ไข (ใครแก้, แก้อะไร, เวลาไหน)
- แสดง history การเปลี่ยนแปลงราคา
- เมื่อแก้ไขราคาทุน ระบบไม่ย้อนหลังคำนวณกำไร-ขาดทุนรายการเก่า

### 1.3 ลบ / ปิดสินค้า
- ใช้ soft-delete เท่านั้น (ไม่ลบข้อมูลจริงออกจาก database)
- ถ้าสินค้ายังมี stock คงเหลือ > 0 จะลบหรือ inactive ไม่ได้
- สถานะสินค้ามี 3 ค่า: active / inactive / discontinued
- สินค้า inactive จะไม่แสดงในหน้าขาย แต่ยังดูประวัติได้

### 1.4 ดูรายการสินค้า
- แสดงตาราง paginated (default 20 รายการ/หน้า)
- เรียงลำดับได้ตาม: ชื่อสินค้า, SKU, stock คงเหลือ, ราคา, วันที่สร้าง
- แสดง stock คงเหลือแบบ real-time
- ไฮไลต์สินค้าที่ stock ต่ำกว่า min ด้วยสีแดง
- Column ราคาทุนแสดงเฉพาะ Admin เท่านั้น

## 2. ค้นหาและกรองสินค้า

### 2.1 ค้นหา (Search)
- ค้นหาแบบ full-text ด้วย: ชื่อสินค้า, รหัสสินค้า (SKU), บาร์โค้ด
- รองรับการพิมพ์บางส่วน (partial match) เช่น "กาแฟ" เจอ "กาแฟดำ", "กาแฟลาเต้"
- ค้นหาแบบไม่สนใจตัวพิมพ์เล็ก/ใหญ่ (case-insensitive)
- แสดงผลลัพธ์แบบ real-time ขณะพิมพ์ (debounce 300ms)

### 2.2 กรอง (Filter)
- กรองตามหมวดหมู่ (รองรับเลือกหลายหมวดพร้อมกัน)
- กรองตามสถานะสินค้า (active / inactive / discontinued)
- กรองตามระดับ stock: สินค้าใกล้หมด / สินค้าหมด / สินค้าปกติ
- กรองตามช่วงราคาขาย (min-max)
- กรองตามวันที่สร้างหรืออัปเดตล่าสุด

### 2.3 บันทึกการค้นหา
- จำการตั้งค่า filter ล่าสุดของแต่ละ user ไว้ (session)
- รีเซ็ต filter ด้วย 1 คลิก

## 3. จัดการหมวดหมู่ (Category Management)

### 3.1 โครงสร้างหมวดหมู่
- รองรับ 2 ระดับ: หมวดหลัก (parent) → หมวดย่อย (child)
- ตัวอย่าง: อาหาร → อาหารแห้ง, เครื่องดื่ม → น้ำอัดลม
- หมวดหมู่มี: ชื่อ, รหัส, สี (สำหรับแสดงผล), ลำดับการแสดง

### 3.2 CRUD หมวดหมู่
- เพิ่ม / แก้ไข / ลบ หมวดหมู่ได้
- ลบ parent category ไม่ได้ถ้ายังมี child อยู่
- ลบ category ที่มีสินค้าอยู่ไม่ได้ (ต้องย้ายสินค้าออกก่อน)
- ย้ายสินค้าข้ามหมวดหมู่ได้แบบ bulk (เลือกหลายรายการพร้อมกัน)

### 3.3 แสดงผลหมวดหมู่
- แสดงเป็น tree structure ในหน้า filter
- แสดงจำนวนสินค้าในแต่ละหมวดหมู่

## 4. สแกนบาร์โค้ด (Barcode Scanning)

### 4.1 รูปแบบที่รองรับ
- EAN-13 (มาตรฐานสากล)
- EAN-8
- QR Code
- Code 128

### 4.2 วิธีสแกน
- สแกนผ่านกล้องของมือถือหรือ tablet
- รองรับเครื่องสแกนบาร์โค้ดแบบ USB/Bluetooth (ป้อนข้อมูลเป็น keyboard input)

### 4.3 พฤติกรรมเมื่อสแกน
- พบสินค้า → เปิดหน้าสินค้าทันที
- ไม่พบสินค้า → แสดงตัวเลือก "เพิ่มสินค้าใหม่" พร้อมกรอกบาร์โค้ดล่วงหน้า
- รองรับ 1 สินค้า มีหลายบาร์โค้ด (เช่น บาร์โค้ดกล่อง vs บาร์โค้ดชิ้น)

### 4.4 บาร์โค้ดไม่ซ้ำกัน
- บาร์โค้ดเดียวกันผูกได้แค่ 1 สินค้าเท่านั้น
- ระบบตรวจสอบและแจ้งเตือนถ้าพยายามเพิ่มบาร์โค้ดซ้ำ

## 5. นำเข้า / ส่งออกข้อมูล (Import / Export)

### 5.1 Import จาก Excel (.xlsx)
- ดาวน์โหลด template Excel ที่มี header ถูกต้อง
- ตรวจสอบข้อมูลก่อน import (validation pass → แสดง preview)
- แสดงรายการ error แยกรายแถวถ้าข้อมูลผิดรูปแบบ
- เลือกได้ว่า import เฉพาะสินค้าใหม่ หรือ update ของเดิมด้วย (upsert)
- รองรับสูงสุด 1,000 รายการต่อครั้ง

### 5.2 Export ออก Excel / CSV
- Export รายการสินค้าทั้งหมด หรือตาม filter ที่เลือกไว้
- เลือก column ที่ต้องการ export ได้
- Admin export ได้ทุก column รวมราคาทุน
- Staff export ได้เฉพาะ column ที่ได้รับอนุญาต

## 6. Product Variants (สินค้าหลาย Variant)

### 6.1 โครงสร้าง Variant
- 1 สินค้าหลัก (parent product) มีหลาย variant ได้
- Variant attributes ตัวอย่าง: สี, ไซส์, รสชาติ, ขนาดบรรจุ
- แต่ละ variant มี SKU, บาร์โค้ด, ราคา, และ stock แยกกัน

### 6.2 การจัดการ Variant
- สร้าง variant ได้จากหน้าแก้ไขสินค้าหลัก
- ตั้งราคาแตกต่างกันแต่ละ variant ได้
- แสดง stock รวมของทุก variant ในหน้าสินค้าหลัก
- เมื่อขาย ตัดสต็อกเฉพาะ variant ที่เลือก

---

## 7. Bundle / Set สินค้า

### 7.1 สร้าง Bundle
- รวมสินค้าหลายชิ้นเป็น 1 bundle (เช่น ชุดของขวัญ)
- กำหนดจำนวนสินค้าย่อย (component) แต่ละรายการใน bundle
- ตั้งราคาขาย bundle แยกจากราคารวมของสินค้าย่อย

### 7.2 การคำนวณ Stock Bundle
- Stock ของ bundle คำนวณจาก component ที่มีน้อยที่สุด
  - ตัวอย่าง: Bundle = A(x2) + B(x1) → A มี 10, B มี 3 → Bundle stock = 3
- เมื่อขาย bundle ตัดสต็อก component ทุกรายการพร้อมกัน
- แจ้งเตือนเมื่อ component ใดมีสต็อกน้อยจนกระทบ bundle

## 8. ประวัติการเปลี่ยนแปลง (Product History)

### 8.1 สิ่งที่บันทึก
- ทุกครั้งที่มีการ: สร้าง, แก้ไข, เปลี่ยนสถานะ, เปลี่ยนราคา
- บันทึก: วันเวลา, ผู้ทำรายการ, ค่าก่อนแก้ไข (before), ค่าหลังแก้ไข (after)

### 8.2 การดู History
- ดูได้จากหน้าแก้ไขสินค้า แท็บ "ประวัติ"
- กรองตามช่วงวันที่หรือประเภทการเปลี่ยนแปลง
- Export history ออก Excel ได้

## 9. Business Rules สรุป

| กฎ | รายละเอียด |
|---|---|
| SKU unique ตลอดกาล | แม้สินค้านั้นจะถูก inactive แล้วก็ตาม ห้ามใช้ SKU ซ้ำ |
| ลบสินค้าต้อง stock = 0 | ต้องทำ stock adjustment เป็น 0 ก่อนถึงจะ inactive ได้ |
| ราคาทุนเห็นเฉพาะ Admin | Staff ทั่วไปไม่เห็นต้นทุนและอัตรากำไร |
| ราคาขาย >= 0 | ยอมให้ราคา 0 ได้ (กรณีสินค้าแถม/ของแจก) |
| บาร์โค้ดไม่ซ้ำในระบบ | 1 บาร์โค้ด ชี้ได้แค่ 1 สินค้าเท่านั้น |
| Min stock trigger alert | stock <= min → ส่งแจ้งเตือนอัตโนมัติทันที |
| ลบ category ที่มีสินค้าไม่ได้ | ต้องย้ายสินค้าออกก่อน |
| Import สูงสุด 1,000 รายการ/ครั้ง | ป้องกัน timeout และ performance issues |

## 10. Non-Functional Requirements

### 10.1 Performance
- หน้ารายการสินค้าโหลดภายใน 2 วินาที (สูงสุด 10,000 รายการ)
- การค้นหาแสดงผลภายใน 500ms
- Import 1,000 รายการเสร็จภายใน 30 วินาที

### 10.2 Usability
- รองรับการใช้งานบนมือถือ (responsive)
- รองรับ keyboard shortcut สำหรับการสแกนบาร์โค้ด
- ฟอร์มมี auto-save draft กันข้อมูลหาย

### 10.3 Security
- ทุก API endpoint ต้องผ่าน authentication
- Log ทุก action ที่เกี่ยวกับสินค้า (audit trail)
- ข้อมูลราคาทุนเข้ารหัสใน database
